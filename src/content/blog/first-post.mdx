## Why This Site Exists

I want `gnaroshi.dev` to work as a public research notebook rather than a polished portfolio template. The site should make it easy to publish:

- technical notes,
- research logs,
- paper summaries,
- project writeups,
- and small implementation records.

The important constraint is maintenance. A good note should be publishable without turning the writing process into a separate project.

## Content Should Stay Close To Git

Markdown and Git are a useful baseline for research writing because the workflow is explicit:

```ts
const note = {
  format: "mdx",
  source: "git",
  goal: "make research progress easy to revisit"
};
```

That shape is intentionally simple. The site should not require a database, CMS, backend, or login flow for the core writing workflow.

## A Small Metric

For paper reading, a useful score is not only whether a paper is finished. A small daily signal could look like:

$$
progress = skim + questions + revisit + implementation
$$

The exact metric matters less than the habit of recording partial progress honestly.

## How This Should Grow

The best version of this site is small, specific, and cumulative. It should collect research questions, implementation notes, and paper-reading observations that remain useful when I return to them later.
