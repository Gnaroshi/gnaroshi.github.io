## Pass 1: Skim And Decide

The first pass answers a small set of questions:

- What problem does the paper claim to solve?
- What is the main contribution?
- Is this paper relevant to my current research map?
- What should I read next if this paper matters?

This pass should be quick. It still counts as progress because deciding relevance is part of research work.

## Pass 2: Understand Structure

The second pass is about the argument structure:

1. Method overview.
2. Core assumptions.
3. Main experiments.
4. Baselines.
5. Failure cases or missing evidence.

At this point, I should be able to explain the paper to another technical reader without going line by line.

## Pass 3: Deep Dive

The third pass is for papers worth deeper effort. This might mean deriving a formula, reading code, reproducing a result, or implementing a small version of the idea.

```python
def reading_depth(pass_number: int, implemented: bool = False) -> int:
    if implemented:
        return 4
    return min(max(pass_number, 0), 3)
```

## What The Tracker Should Reward

The tracker should reward consistency, questions, revisits, and implementation attempts. A partial note with a sharp question is often more useful than an empty promise to read the paper later.
