## 이 사이트가 필요한 이유

`gnaroshi.dev`를 완성된 포트폴리오보다 공개 연구 노트에 가깝게 쓰고 싶습니다. 이곳에는 다음과 같은 기록을 부담 없이 남깁니다.

- 기술 노트
- 연구 기록
- 논문 요약
- 프로젝트 회고
- 작은 구현 시도

가장 중요한 조건은 꾸준히 관리할 수 있어야 한다는 점입니다. 노트 하나를 공개하기 위해 글쓰기와 별개의 프로젝트를 시작해야 한다면 오래 유지하기 어렵습니다.

## 콘텐츠를 Git 가까이에 두기

Markdown과 Git은 연구 글을 남기기에 충분히 명확한 출발점입니다.

```ts
const note = {
  format: "mdx",
  source: "git",
  goal: "make research progress easy to revisit"
};
```

핵심 글쓰기 흐름에는 데이터베이스나 CMS, 백엔드, 로그인 절차가 필요하지 않습니다. 파일을 쓰고 검토한 뒤 커밋하는 단순한 구조를 유지합니다.

## 작은 지표

논문 읽기에서 중요한 것은 완독 여부만이 아닙니다. 하루의 작은 진전은 다음처럼 기록할 수 있습니다.

$$
progress = skim + questions + revisit + implementation
$$

정확한 수식보다 중요한 것은 부분적인 진전도 솔직하게 남기는 습관입니다.

## 앞으로 쌓아 갈 것

이 사이트는 작고 구체적인 기록이 차곡차곡 이어질 때 가장 유용합니다. 나중에 다시 돌아왔을 때도 쓸모 있는 연구 질문과 구현 메모, 논문 읽기 관찰을 남기려 합니다.
