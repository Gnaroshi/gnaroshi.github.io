## Why Markdown

Markdown keeps the writing path short. A note can start as an imperfect record:

```md
## Question

Why does this method fail under a different benchmark?

## Next Step

Check whether the implementation depends on an unstated preprocessing step.
```

That is enough to preserve context for the next session.

## Why Git

Git gives research notes a useful history:

- what changed,
- when the direction shifted,
- which implementation attempt matched a paper note,
- and which questions stayed unresolved.

This is especially useful for a public site because every published change has an explicit commit.

## A Practical Loop

A sustainable workflow can stay small:

1. Read or build something.
2. Write the smallest useful note.
3. Link it to a paper, project, or question.
4. Commit it.
5. Revisit it when a pattern appears.

The point is not to make a perfect knowledge base. The point is to make research state recoverable.
